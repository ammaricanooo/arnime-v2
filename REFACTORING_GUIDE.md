## 🔄 FILE ORGANIZATION IMPROVEMENTS

### ✅ What Changed

1. **lib/types.ts** - Centralized TypeScript definitions
   - 📍 Replaces scattered interface definitions
   - 🎯 Single source of truth for data structures
   - 📦 Easier to maintain and refactor

2. **lib/constants.ts** - All magic strings & config
   - 📍 API endpoints, timeouts, cache keys
   - 📍 Error messages, storage keys
   - 🎯 Easy to update configs globally

3. **lib/api.ts** - Centralized API client
   - 📍 Replaces `lib/fetchJson.ts`
   - ✨ Added retry logic, caching, timeout handling
   - ✨ Comprehensive error handling with custom ApiError class
   - 🎯 Better composability and reusability

4. **lib/seo.ts** - NEW metadata generators
   - 📍 Generates OpenGraph, Twitter cards
   - 📍 Schema.org structured data (JSON-LD)
   - 📍 Breadcrumbs, Organization schema
   - 🎯 Better Google indexing & rich snippets

5. **lib/sitemap.ts** - NEW SEO utilities
   - 📍 Generates XML sitemap
   - 📍 Generates robots.txt
   - 📍 Manages crawl priorities
   - 🎯 Better SEO discovery & indexing

6. **Folder Structure Reorganization**
   - 📍 `components/ui/` - Base UI components
   - 📍 `components/layouts/` - Layout components
   - 📍 `components/grid/` - Grid/list components
   - 📍 `lib/hooks/` - Custom React hooks
   - 📍 `lib/utils/` - Helper functions
   - 📍 `styles/` - Centralized CSS
   - 🎯 Clear separation of concerns

## 🚀 PERFORMANCE OPTIMIZATIONS

### 1. Bundle Size Reduction
- ✅ Code splitting by route (automatic with Next.js)
- ✅ Webpack configured to separate vendor chunks
- ✅ Tree shaking enabled (unused code removed)
- ✅ Image optimization with AVIF/WebP formats

### 2. Server vs Client Components
```tsx
// ✅ GOOD: Server component with metadata
// app/anime/[slug]/page.tsx
export async function generateMetadata({ params }) {
  const anime = await fetchJson(`/api/detail/${params.slug}`, {
    cacheKey: `anime-${params.slug}`,
  })
  return generateAnimeMetadata(anime)
}

export default async function AnimePage({ params }) {
  const anime = await fetchJson(`/api/detail/${params.slug}`)
  return <AnimeClientPage anime={anime} />
}

// ❌ AVOID: Client component doing fetching
// This causes waterfalls and poor performance
'use client'
export default function Page() {
  const [anime, setAnime] = useState(null)
  useEffect(() => {
    // This is slower - fetch happens client-side
    fetchJson(url).then(setAnime)
  }, [])
}
```

### 3. Caching Strategy
```typescript
// lib/constants.ts
export const REVALIDATION_TIMES = {
  NEVER: false,              // Cache forever (ISR disabled)
  INSTANT: 0,                // Revalidate on every request
  VERY_SHORT: 60,            // 1 minute
  SHORT: 300,                // 5 minutes
  MEDIUM: 3600,              // 1 hour
  LONG: 86400,               // 24 hours
  VERY_LONG: 604800,         // 7 days
}

// Usage in server components
const anime = await fetchJson(url, {
  cacheKey: `anime-${slug}`,  // Cache key for in-memory cache
  next: { revalidate: REVALIDATION_TIMES.LONG } // ISR revalidation
})
```

### 3. Sitemap & Robots.txt
```typescript
// app/sitemap.ts
import { generateSitemap, PAGE_PRIORITY, PAGE_CHANGE_FREQUENCY } from '@/lib/sitemap'

export default async function sitemap() {
  const anime = await fetchJson('/api/anime/list')

  const urls = [
    {
      loc: `${SEO_DEFAULTS.SITE_URL}/`,
      changefreq: 'daily' as const,
      priority: PAGE_PRIORITY.CRITICAL,
    },
    ...anime.map(item => ({
      loc: `${SEO_DEFAULTS.SITE_URL}/anime/${item.slug}`,
      changefreq: 'weekly' as const,
      priority: PAGE_PRIORITY.MEDIUM,
    })),
  ]

  return urls
}

// app/robots.ts
import { generateRobotsTxt } from '@/lib/sitemap'

export default function robots() {
  return {
    rules: [
      { userAgent: '*', allow: '/', disallow: ['/admin', '/api'] },
    ],
    sitemap: `${SEO_DEFAULTS.SITE_URL}/sitemap.xml`,
  }
}
```

## 💡 BEST PRACTICES

### 1. API Usage
```typescript
// ✅ GOOD: Using centralized API client
import { fetchJson } from '@/lib/api'

const anime = await fetchJson('/api/detail/slug', {
  cacheKey: 'anime-detail-slug',
  retries: 2,
  timeout: 10000,
})

// ❌ AVOID: Raw fetch without error handling
const res = await fetch(url)
const data = await res.json() // Will crash if not valid JSON
```

### 2. Type Safety
```typescript
// ✅ GOOD: Using centralized types
import { AnimeDetail, ComicCard } from '@/lib/types'

interface Props {
  anime: AnimeDetail
  comics: ComicCard[]
}

// ❌ AVOID: Inline any type
function Component(props: any) {} // No type safety
```

### 3. Constants Management
```typescript
// ✅ GOOD: Using centralized constants
import { API_ENDPOINTS, ERROR_MESSAGES, REVALIDATION_TIMES } from '@/lib/constants'

const anime = await fetchJson(API_ENDPOINTS.OTAKUDESU.DETAIL)

// ❌ AVOID: Magic strings scattered everywhere
const anime = await fetchJson('https://api.ammaricano.my.id/api/otakudesu/detail')
```

### 4. Component Organization
```typescript
// ✅ GOOD: Clear export structure
export interface Props { ... }
export interface State { ... }
export default function Component(props: Props) { ... }

// ❌ AVOID: Anonymous functions & unclear exports
export default (props) => <div>...</div>
```

## 📚 MIGRATION GUIDE

### From Old fetchJson to New API

**Before:**
```typescript
const res = await fetch(url)
if (!res.ok) throw new Error('Failed')
const data = await res.json()
```

**After:**
```typescript
import { fetchJson } from '@/lib/api'
const data = await fetchJson(url, {
  cacheKey: 'unique-key',
  retries: 2,
  timeout: 10000,
})
```

### From Inline Types to Centralized Types

**Before:**
```typescript
interface AnimeDetail {
  title: string
  slug: string
  // ... 20 more fields
}
```

**After:**
```typescript
import { AnimeDetail } from '@/lib/types'
// Use throughout the app
```
